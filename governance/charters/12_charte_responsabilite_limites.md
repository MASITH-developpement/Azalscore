## 4. LIMITES DU SYSTÈME

### 4.1 Ce Qu'AZALSCORE EST

```
AZALSCORE EST:

✅ Un outil d'aide à la décision
✅ Un système de gestion intégré (ERP)
✅ Une plateforme de pilotage
✅ Un assistant augmenté par IA
✅ Un dépositaire de données métier
```

### 4.2 Ce Qu'AZALSCORE N'EST PAS

```
AZALSCORE N'EST PAS:

❌ Un conseiller juridique
❌ Un conseiller fiscal
❌ Un conseiller financier
❌ Un expert-comptable
❌ Un commissaire aux comptes
❌ Un décideur autonome
❌ Une garantie de résultat
```

### 4.3 Limites Fonctionnelles

| Domaine | Limite |
|---------|--------|
| Comptabilité | Outil de saisie, pas de certification |
| Fiscalité | Aide au calcul, pas de conseil fiscal |
| Paie | Calcul indicatif, validation humaine requise |
| Trésorerie | Prévisions, pas de garantie |
| IA | Suggestions, pas de décisions |

## 5. EXCLUSIONS DE RESPONSABILITÉ

### 5.1 Exclusions Absolues

```
L'ÉDITEUR N'EST PAS RESPONSABLE DE:

1. DÉCISIONS PRISES PAR L'UTILISATEUR
   Le système propose, l'humain décide et assume.

2. DONNÉES SAISIES PAR LE CLIENT
   Le client est responsable de l'exactitude de ses données.

3. USAGE NON CONFORME
   L'usage contraire aux chartes exclut toute responsabilité.

4. FORCE MAJEURE
   Événements hors de contrôle (catastrophe, cyberattaque étatique...).

5. CONSEIL PROFESSIONNEL
   Les informations ne remplacent pas un conseil professionnel.
```

### 5.2 Limitation de Responsabilité

```
EN CAS DE DOMMAGE AVÉRÉ:

La responsabilité de l'éditeur est limitée à:
- Dommages directs uniquement
- Plafond = montant des 12 derniers mois d'abonnement
- Exclusion des dommages indirects, manque à gagner

EXCLUS:
- Pertes d'exploitation
- Perte de données non sauvegardées
- Dommages consécutifs
- Atteinte à l'image
```

## 6. RESPONSABILITÉ HUMAINE

### 6.1 Principe Fondamental

```
RÈGLE: L'humain est responsable de ses décisions.

AZALSCORE fournit:
- Des données
- Des analyses
- Des suggestions
- Des alertes

AZALSCORE ne fournit PAS:
- Des décisions
- Des garanties de résultat
- Des conseils professionnels certifiés
```

### 6.2 Validation Obligatoire

| Type d'Action | Validation Requise | Responsable |
|---------------|-------------------|-------------|
| Paiement | Humaine | Utilisateur validateur |
| Engagement contractuel | Humaine | Dirigeant |
| Suppression données | Humaine | Selon criticité |
| Alerte RED | Humaine | Dirigeant uniquement |

### 6.3 Non-Substitution

```
AZALSCORE NE SE SUBSTITUE PAS À:

- Un expert-comptable pour la certification des comptes
- Un avocat pour les conseils juridiques
- Un notaire pour les actes authentiques
- Un commissaire aux comptes pour l'audit légal
- Un conseiller fiscal pour l'optimisation fiscale
```

## 7. RÈGLES DE VALIDATION

### 7.1 Principe du Double Contrôle

```
RÈGLE: Les actions critiques nécessitent une double validation.

Créateur ≠ Validateur pour:
- Paiements au-delà du seuil
- Modifications de prix
- Remises exceptionnelles
- Créations d'utilisateurs
```

### 7.2 Traçabilité des Validations

```
Chaque validation enregistre:
- Identité du validateur
- Date et heure
- Contexte de validation
- Éléments vérifiés
```
