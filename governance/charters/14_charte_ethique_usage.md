## 4. REFUS DES DARK PATTERNS

### 4.1 Définition

```
DARK PATTERNS = Techniques de design manipulatrices
visant à tromper ou forcer l'utilisateur à faire
quelque chose qu'il ne souhaite pas.
```

### 4.2 Patterns Interdits

| Pattern | Description | Interdit |
|---------|-------------|----------|
| **Bait and Switch** | Attirer puis changer | ✅ |
| **Confirmshaming** | Culpabiliser pour refus | ✅ |
| **Disguised Ads** | Pubs déguisées en contenu | ✅ |
| **Forced Continuity** | Renouvellement caché | ✅ |
| **Friend Spam** | Spam via contacts | ✅ |
| **Hidden Costs** | Frais cachés | ✅ |
| **Misdirection** | Détourner l'attention | ✅ |
| **Price Comparison** | Comparaisons trompeuses | ✅ |
| **Privacy Zuckering** | Piéger sur la vie privée | ✅ |
| **Roach Motel** | Facile d'entrer, dur de sortir | ✅ |
| **Sneak into Basket** | Ajouts non sollicités | ✅ |
| **Trick Questions** | Questions pièges | ✅ |

### 4.3 Exemples Concrets

```
❌ INTERDIT:

"Êtes-vous SÛR de vouloir annuler ?
Vous allez PERDRE tous vos avantages !"
[Non, je veux garder mes avantages] [Oui, annuler quand même]

✅ CORRECT:

"Confirmer l'annulation de votre abonnement ?"
[Annuler l'abonnement] [Garder l'abonnement]
```

```
❌ INTERDIT:

Newsletter pré-cochée lors de l'inscription

✅ CORRECT:

Newsletter non cochée par défaut
```

## 5. IA EXPLICABLE

### 5.1 Principe

```
RÈGLE: L'IA doit être compréhensible.

L'utilisateur doit pouvoir comprendre:
- Pourquoi une suggestion est faite
- Sur quelles données elle se base
- Quel est le niveau de confiance
- Quelles sont les limites
```

### 5.2 Exigences d'Explicabilité

| Aspect | Exigence |
|--------|----------|
| Raison | Expliquer POURQUOI cette suggestion |
| Données | Indiquer les données utilisées |
| Confiance | Afficher le score de confiance |
| Limites | Mentionner ce que l'IA ne sait pas |
| Alternative | Proposer de voir d'autres options |

### 5.3 Exemple

```
✅ BON EXEMPLE:

"Prévision trésorerie : 15 000 €
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
Confiance: 85%

Basé sur:
• Historique 12 derniers mois
• Factures en attente de paiement
• Charges récurrentes connues

Limites:
• Ne prend pas en compte les événements exceptionnels
• Basé sur les tendances passées

[Voir détails] [Ajuster les paramètres]"
```

### 5.4 Interdictions IA

```
❌ INTERDIT:

- IA présentée comme infaillible
- Décision IA sans explication
- IA masquant ses incertitudes
- IA influençant les décisions sans transparence
```

## 6. CONSENTEMENT ÉCLAIRÉ

### 6.1 Principe

```
RÈGLE: Le consentement doit être libre, spécifique et éclairé.

Libre: Pas de pression, pas de conséquence négative au refus
Spécifique: Pour un usage précis, pas générique
Éclairé: Information complète et compréhensible
```

### 6.2 Collecte de Données

```
AVANT TOUTE COLLECTE:

✅ Expliquer QUOI est collecté
✅ Expliquer POURQUOI c'est collecté
✅ Expliquer COMMENT c'est utilisé
✅ Donner le choix de refuser
✅ Permettre de changer d'avis
```

### 6.3 Opt-in vs Opt-out

```
RÈGLE: Privilégier l'opt-in explicite.

Opt-in obligatoire pour:
- Communications marketing
- Partage de données avec tiers
- Fonctionnalités IA avancées
- Analyses comportementales

Opt-out acceptable pour:
- Cookies fonctionnels
- Statistiques anonymes internes
```
